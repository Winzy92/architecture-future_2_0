# Разделение на домены

Разделение на домены сделаем по бизнес целям. У каждого домена будет своя зона ответсвенности и свой набор данных (БД).

## Домены
1. Medical Domain. Внутри хранится операционная деятельность клиник, записи на прием, расписания врачей. Также внутри домена будет 2 области хранения данных, 1 - чувствительные данные пациентов, в шифрованном виде, доступ к ним ограничен, и 2 - обезличенные данные для аналитики.
2. Fintech Domain. Банковские операции, счета, кредиты, операционка, в домене так же будет 2 области, 1 - чувствительные данные, доступ ограничен, настроено шифрование, 2 - обезличенные данные для аналитики.
3. AI Domain. Модели и результаты моделей.
4. Corporate Domain. Корпоративный домен, данные компании, внутренняя документация, операционная деятельность. В домене делаем так же 2 области хранения - 1-я чувствительные данные (защита чувствительных данных компании), 2 - обезличенные данные для аналитики и отчетов.
5. Platform Domain. По сути это инфраструктурный слой взаимодействия, тут будет kafka, Api Gateway и тп.

## Преимущества разделения
1. В текущей реализации любые новые интеграции вели к рискам сломать что то существующее в DWH. В новой интепретации новые интеграции будут реализовываться через kafka и затрагивать существующие БП. Мы снизим время необходимое на вывод новой фичи в прод.
2. Появляется быстрая аналитика, у каждого домена есть область с данными для быстрого построения отчетов.
3. Соблюдения ФЗ по обращению с чувствительными данными, храним раздельно, шифруем, обезличиваем, RBAC/ABAC.
4. Независимость доменов, повышает наблюдаемость системы, легче ее поддеживать.
5. Легче масшатабировать систему, можно подключать внешние сервисы потенциальных патрнеров.
6. Рост стабильности и отказоустойчивости приведет к росту репутации компании.
