# Планируемые изменения и эффект для бизнеса
1. Data Portal - это витрина данных, необходима для self-аналитки. Аналитики быстро получают необходимую информацию, решения принимаются быстрее, убирается связака IT и аналитики.
2. Реализация Data Platform - мы отказываемся от монолитного DWH, позволяет масштабироваться. Рост быстродействия.
3. Data Mesh - раздение на домены, уменьшаем time-to-market, упрощает рост проекта (реализация нового функционала).
4. Защита чувствительных данных - сюда же отнесем RBAC/ABAC. Выполняем требования регулятора, снижаем риски утечки данных.
5. Внедрение event-driven - отказ от ESB позволит проще расти и ингрерировать новые сервисы.


   | Цель бизнеса       | Решение                                                       |
   | ------------------ |---------------------------------------------------------------|
   | Быстрая отчётность | Data Products + Lakehouse                                     |
   | Масштабирование    | Cloud + Data Mesh                                             |
   | Новые бизнесы      | Event-driven                                                  |
   | Безопасность       | Разделение на домены + внедрение защиты чувствительных данных |
   | Снижение затрат    | отказ от legacy                                               |
